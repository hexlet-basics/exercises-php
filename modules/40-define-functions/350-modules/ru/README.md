Пока программа маленькая, весь код можно хранить в одном файле. Такой подход удобен для простых примеров и небольших задач. Но со временем программа начинает расти. Когда кода становится много, ориентироваться в одном файле становится сложно. Между тем реальные приложения состоят из десятков тысяч строк (как минимум) и сотен файлов.

Чтобы разделить программу на отдельные логические части, код раскладывают по файлам. Во многих языках такие файлы называются модулями. В PHP модуль — это обычный файл с кодом и расширением `.php`. А язык предоставляет механизм для обращения к функциям и константам из других файлов — подключение файлов.

## Как подключить файл

Инструкция `require` подключает указанный файл и делает его содержимое доступным:

```php
<?php
// helpers.php

function sayHi()
{
    print_r('Hi!');
}
```

```php
<?php
// Solution.php

require 'helpers.php';

sayHi(); // => Hi!
```

После подключения все функции из файла `helpers.php` доступны так, как будто они были определены прямо в текущем файле.

Кстати, вы уже видели этот механизм в действии. Автоматизированные тесты в этом курсе подключают файл с вашим решением строкой `require 'Solution.php';` — и после этого могут вызывать функции, которые вы определили.

## require, include и require_once

У `require` есть близнец — инструкция `include`. Разница проявляется, когда подключаемый файл не найден: `include` выведет предупреждение, и программа продолжит работу, а `require` остановит выполнение с ошибкой. Если без подключаемого кода программа не имеет смысла, используют `require`.

Есть и третий вариант — `require_once`. Он подключает файл только один раз, даже если инструкция встречается в программе многократно. Это важно для файлов с функциями: повторное определение функции с тем же именем приводит к ошибке.

## Встроенные функции доступны без подключения

Стандартная библиотека PHP устроена проще, чем во многих других языках: встроенные функции доступны в любом месте программы сразу, без какого-либо подключения. Например, функции округления: `floor()` округляет число вниз до ближайшего целого, `ceil()` округляет вверх:

```php
<?php

print_r(floor(7.9)); // => 7
print_r(ceil(7.1));  // => 8
print_r(ceil(7.0));  // => 7
```

Разница заметна, когда число не целое. `floor(7.9)` дает 7, а не 8, потому что 7 — ближайшее целое снизу.

Встроенные функции, конечно, доступны и в теле своих функций:

```php
<?php

// Вычисляет количество поездок, необходимых для перевозки всех предметов
function tripsNeeded(int $items, int $capacity): int
{
    return (int) ceil($items / $capacity);
}

print_r(tripsNeeded(10, 3)); // => 4
```

Обратите внимание: `ceil()` возвращает число типа `float`, поэтому результат приводится к `int` — так, как мы делали в уроке про преобразование типов.

## Конфликты имен

В больших программах разные части кода могут содержать функции с одинаковыми названиями. В PHP нельзя определить две функции с одним именем в одном пространстве — программа завершится с ошибкой `Cannot redeclare function`.

Для борьбы с конфликтами имен в PHP существуют **пространства имен** (namespaces). Вы видели их в начале каждого задания этого курса — строка `namespace HexletBasics\...;` как раз объявляет пространство имен. Благодаря им мы можем создавать сколько угодно функций с одинаковыми именами, главное чтобы они находились в разных пространствах. Это работает даже со встроенными функциями:

```php
<?php

namespace App\Custom;

function floor($number)
{
    return 'custom floor';
}

print_r(floor(3.7));  // => custom floor
print_r(\floor(3.7)); // => 3
```

Внутри пространства `App\Custom` имя `floor` ссылается на нашу функцию. Встроенная функция при этом никуда не делась — она доступна по полному имени `\floor()`, где обратный слеш означает глобальное пространство. Это особенно важно при использовании сторонних библиотек, потому что разные разработчики могут выбирать одинаковые имена для функций. Подробно пространства имен изучаются дальше, на Хекслете.
