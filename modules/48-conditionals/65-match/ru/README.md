Выражение `match` представляет собой специализированную версию `if`, которую создали для особых ситуаций. Оно появилось в PHP 8, и его нужно использовать там, где есть цепочка `if else` с проверками на равенство:

```php
<?php

if ($status === 'processing') {
    $statusText = 'Заказ в обработке';
} elseif ($status === 'paid') {
    $statusText = 'Заказ оплачен';
} elseif ($status === 'new') {
    $statusText = 'Новый заказ';
} else {
    $statusText = 'Неизвестный статус';
}
```

У этой составной проверки есть одна отличительная черта. Каждая ветка здесь является проверкой значения переменной `$status`. Выражение `match` позволяет записать этот код короче и выразительнее:

```php
<?php

$statusText = match ($status) {
    'processing' => 'Заказ в обработке', // $status === 'processing'
    'paid' => 'Заказ оплачен',           // $status === 'paid'
    'new' => 'Новый заказ',              // $status === 'new'
    default => 'Неизвестный статус',     // else
};
```

```text
match (значение) {
  │
  ├── 'a' → результат 1
  ├── 'b' → результат 2
  ├── 'c' → результат 3
  └── default → результат по умолчанию
}
```

С точки зрения количества элементов `match` является сложной конструкцией. Внешнее описание включает ключевое слово `match` и переменную, по значениям которой `match` будет выбирать поведение. Внутри расположены ветки, каждая из которых описывает результат для одного из значений переменной. Каждая ветка соответствует `if` в примере выше. При этом `default` является особой ситуацией, которая соответствует ветке `else` в условных конструкциях.

Главное отличие `match` от обычного `if` в том, что это **выражение**. Вся конструкция вычисляется в значение, поэтому ее результат можно сразу присвоить переменной или вернуть из функции. Никаких `break` не нужно: выполняется ровно одна ветка, «проваливание» здесь невозможно.

Внутри `match` допустим только тот синтаксис, который показан выше. Слева от стрелки `=>` указывается значение для сравнения, справа — выражение, которое станет результатом. Если одному результату соответствует несколько значений, они перечисляются через запятую:

```php
<?php

$type = match ($size) {
    'xs', 's' => 'small',
    'm' => 'medium',
    'l', 'xl' => 'large',
};
```

Еще две важные детали:

* `match` сравнивает значения строго, как оператор `===` — автоматического преобразования типов не происходит.
* Если ни одна ветка не совпала и `default` не указан, программа завершится с ошибкой `UnhandledMatchError`.

Часто результат `match` нужно вернуть из функции, которая его содержит. Для этого перед `match` ставится обычный `return`:

```php
<?php

function countItems(int $count): ?string
{
    return match ($count) {
        1 => 'one',
        2 => 'two',
        default => null,
    };
}

print_r(countItems(2)); // => two
```

Выражение `match` хоть и встречается в коде, но технически всегда можно обойтись без него. Ключевая польза при его использовании в том, что он лучше выражает намерение программиста, когда нужно проверять конкретные значения переменной. Такой код читать легче, чем блоки `elseif`.
