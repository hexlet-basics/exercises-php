Изучение нового языка программирования традиционно начинается с программы 'Hello, World!'. Это простая программа, которая выводит приветствие на экран и знакомит с синтаксисом и структурой нового языка.

```text
Hello, World!
```

![Hello World](./assets/hello-world.png)

Этой традиции уже больше сорока лет, и мы тоже начнем с нее. В первом уроке мы напишем программу `Hello, World!`. На PHP эта программа выглядит так:

```php
<?php

print_r('Hello, World!');
```

Строка `<?php` в начале — специальный тег, который сообщает, что дальше идет код на PHP. Подробнее о нем мы поговорим в следующем уроке.

Команда `print_r()` выводит на экран текст, указанный в скобках. В конце ставится точка с запятой `;` — так в PHP завершается каждая команда. Вместо примера можно написать любой другой текст.

```php
<?php

print_r('Хекслет - школа программирования');
```

Команда остается той же, меняется только содержимое скобок. Чтобы программа понимала, что это именно текст, он заключается в кавычки. Можно использовать одинарные `'...'` или двойные `"..."`, но открывающая и закрывающая кавычки должны совпадать.

```php
<?php

print_r("Хекслет - школа программирования");
```

В PHP принято использовать одинарные кавычки для строк. Если внутри строки есть апостроф, одинарные кавычки сломают синтаксис, поэтому в таких случаях используют двойные.

```php
<?php

print_r("it's PHP"); // апостроф внутри, поэтому двойные кавычки
```

## Способы вывода на экран

Кроме `print_r()`, в PHP есть еще одна команда вывода — `echo`:

```php
<?php

echo 'Hello, World!';
// => Hello, World!
```

Для удобства мы будем показывать результат запуска строк кода так: `// => РЕЗУЛЬТАТ`.

В простых ситуациях между `echo` и `print_r()` нет разницы — можно использовать любую команду. А там, где на экран нужно выводить не просто числа или строки, а, например, массивы, `echo` не сможет этого сделать, а `print_r()` все выведет.

## Значение символов

Код состоит из команд, и каждая из них должна быть написана в определенной форме. Помимо букв, в коде важны кавычки `'` и `"`, скобки `()` и знаки препинания. Пропущенный или перепутанный знак приведет к тому, что программа не запустится. Попробуйте определить, какая ошибка допущена в каждой из строк?

```php
<?php

print_r("it's PHP"
print_r(it's PHP")
prin_r("it's PHP")
print_r('it's PHP")
print_r["it's PHP"];
```

Даже небольшое отличие, например одна лишняя буква или другой знак, может привести к тому, что программа не будет работать. Это относится и к регистру, то есть к различию между большими и маленькими буквами. Если в обычном тексте `Привет` и `привет` выглядят одинаково, то для программы это разные тексты. Имена функций PHP прощают разный регистр, но имена переменных и многие другие конструкции, с которыми мы встретимся дальше, — нет. Поэтому надежнее всего набирать код в точности так, как в примерах.

## Где практиковаться

Теория усваивается лучше, когда параллельно запускаешь код и видишь результат. Для этого подходит онлайн-интерпретатор PHP, где код выполняется прямо в браузере. Все, что встречается в уроке, стоит пробовать [в онлайн-интерпретаторе PHP](https://3v4l.org/).

Как это работает технически? Любой написанный код передается в интерпретатор PHP, который этот код выполняет и выводит на экран результат его работы.

```text
Код               Интерпретатор        Экран
┌────────────┐     ┌─────────────┐     ┌──────────────┐
│ print_r(…) │ ──→ │     PHP     │ ──→ │ Hello, World!│
└────────────┘     └─────────────┘     └──────────────┘
```
